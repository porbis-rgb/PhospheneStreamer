Build a small, minimal‑credit virtual implant firmware simulator that integrates directly with my existing Phosphene Streamer project.

Goal:  
Simulate how an implant would receive stimulation commands generated by Phosphene Streamer (electrode_id, amplitude, duration_us), apply simple safety checks, and “activate” a virtual electrode array. This is only a software simulation for research and educational purposes.

Integration requirement:

Assume Phosphene Streamer outputs a Python dict or JSON object like:
{ "electrode_id": int, "amplitude": int, "duration_us": int }

Create a single Python function process_stim_packet(packet) that Phosphene Streamer can call directly.

Keep everything extremely small and in one place:

One src/ folder

Files: electrodes.py, firmware_sim.py, packets.py, main.py

No extra abstractions, no unnecessary classes

Core features (minimal versions):

Electrode array

Represent an 8x8 grid using a simple list or dict.

Store last amplitude + last activation time.

Stimulation command

A tiny dataclass StimCommand(electrode_id, amplitude, duration_us).

Function apply_command(cmd) updates the electrode state.

Safety checks (very simple)

Reject amplitude > MAX_SAFE_AMPLITUDE.

Add a basic “thermal” rule: if the same electrode is stimulated too frequently, print a warning.

Log a “charge balancing” pulse by printing it (no real physics).

Packet parsing

Convert the dict/JSON from Phosphene Streamer into a StimCommand.

Visualization (minimal)

Print an ASCII 8x8 grid showing activation intensity (0–9).

No HTML, no JS, no Canvas.

API or CLI (choose the simplest)

Provide a CLI in main.py that runs a few sample stimulations.

Also expose a single Python function simulate_from_streamer(packet) so Phosphene Streamer can import and call it.

Tiny tests

2–3 pytest tests verifying safety checks and packet parsing.

Constraints:

Keep the entire project extremely small.

Avoid unnecessary comments or boilerplate.

Focus on making the simulator callable directly from Phosphene Streamer with no extra setup.

Produce the full scaffold and implementation in one response to minimize credit usage.
